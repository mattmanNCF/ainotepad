**Step 3 — Create build/make-icon.mjs (build script, not shipped in app):**
```javascript
import sharp from 'sharp'
import { mkdirSync } from 'fs'
import { join } from 'path'

// Inline SVG buffer — avoids file I/O dependency issues with librsvg on Windows
const svgBuffer = Buffer.from(`<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 256 256" width="256" height="256">
  <rect x="20" y="15" width="216" height="226" rx="30" ry="30" fill="#1a1a2e"/>
  <polygon points="186,15 236,15 236,65" fill="#2d2d4e"/>
  <line x1="186" y1="15" x2="236" y2="65" stroke="#4a4a7a" stroke-width="3"/>
  <rect x="45" y="85" width="125" height="18" rx="9" ry="9" fill="#6c6cad"/>
  <rect x="45" y="119" width="166" height="18" rx="9" ry="9" fill="#6c6cad"/>
  <rect x="45" y="153" width="105" height="18" rx="9" ry="9" fill="#6c6cad"/>
  <circle cx="45" cy="192" r="9" fill="#8080cc"/>
</svg>`)

mkdirSync('build/icon-work', { recursive: true })
await sharp(svgBuffer).resize(1024, 1024).png().toFile('build/icon-work/icon-1024.png')
console.log('Generated build/icon-work/icon-1024.png')
```

Run the script:
```bash
node build/make-icon.mjs
```

**Step 4 — Run electron-icon-builder to produce .ico and .icns:**
```bash
./node_modules/.bin/electron-icon-builder --input=build/icon-work/icon-1024.png --output=build/icon-work/out --flatten
```

This produces:
- `build/icon-work/out/icons/icon.ico` — multi-size ICO (16–256px)
- `build/icon-work/out/icons/mac/icon.icns` — macOS ICNS
- `build/icon-work/out/icons/png/256x256.png` — PNG fallback

**Step 5 — Copy outputs to build/ (replacing Electron defaults):**
```bash
cp build/icon-work/out/icons/icon.ico build/icon.ico
cp build/icon-work/out/icons/mac/icon.icns build/icon.icns
cp build/icon-work/out/icons/png/256x256.png build/icon.png
```

**Step 6 — Clean up work directory:**

cp build/icon-work/icon-1024.png build/icon-1024.png
rm -rf build/icon-work build/make-icon.mjs

Keep `build/icon-source.svg` and `build/icon-1024.png` as source of record.
