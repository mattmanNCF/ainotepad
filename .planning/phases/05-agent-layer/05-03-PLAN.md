
**Test 2 — tools/list (4 tools registered):**
```bash

  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}'
```
PASS criteria: Response contains all 4 names: `get_recent_notes`, `search_notes`, `get_wiki_page`, `list_wiki_pages`.

**Test 3 — get_recent_notes (no params → default 20):**
```bash

  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"get_recent_notes","arguments":{}}}'
```
PASS criteria: Response contains `"content"` array with `"type":"text"` item. Inner text is valid JSON (array, possibly empty if no notes exist).

**Test 4 — search_notes with safe query:**
```bash

  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":4,"method":"tools/call","params":{"name":"search_notes","arguments":{"query":"the","limit":5}}}'
```
PASS criteria: Response contains `"content"` array. Inner text is valid JSON (array, empty OK if no notes indexed).

**Test 5 — search_notes with malformed FTS5 query (error resilience):**

**Test 6 — list_wiki_pages:**
```bash

  -H "Content-Type: application/json" \

```
PASS criteria: Response contains `"content"` with valid JSON array (empty `[]` OK if KB has no pages).

**Test 7 — get_wiki_page for nonexistent page (error handled gracefully):**
