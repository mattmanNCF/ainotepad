Digest generation flow (CRITICAL — word cloud built in main process, not worker):
1. digestScheduler.ts: query `SELECT tags, submitted_at FROM notes WHERE hidden=0 AND ai_state='complete' AND submitted_at >= ?`
2. Parse tags JSON, count frequencies → build [{text, value}] array (cap at 50 words)
3. Build stats: { noteCount, topTags: string[], mostActiveDay: string }
4. Query last digest date: `SELECT generated_at FROM digests WHERE period='daily' ORDER BY generated_at DESC LIMIT 1`
5. Check elapsed hours. If < 20: skip. If >= 20: post digest-task.
6. Post to workerPort: { type: 'digest-task', period: 'daily', periodStart: ISO, wordCloudData: JSON, stats: JSON, braveKey?: string }
7. Worker generates narrative text using AI (local model or frontier API)
8. Worker optionally calls Brave Search if braveKey provided
9. Worker posts back: { type: 'digest-result', period, periodStart, wordCloudData, narrative, stats, generatedAt }
10. aiOrchestrator result handler INSERTs into digests, pushes 'digest:updated' to renderer

Brave Search API (from research):
- Endpoint: https://api.search.brave.com/res/v1/web/search
- Header: X-Subscription-Token: <key>
- Query param: q=<encoded query>, count=5
- Skip entirely if braveKey is absent or empty string
- Used only for novelty-type insights in notes, and for general market research context in digests
