**Pre-test prep (Matt performs once):**

A. Uninstall the existing Notal if installed: Start -> Settings -> Apps -> Notal -> Uninstall.
B. Run the new installer: dist/Notal Setup *.exe. Default install location.
C. Launch Notal. Confirm it boots without errors.
D. Open Settings -> Integrations. Note the current state of the Google Calendar section (should be whatever Matt's Phase 11 state is — connected or not).
E. If Phase 11 showed "connected", DISCONNECT now (to force the MOB-AUTH-01 incremental consent flow on reconnect).
F. On iPhone, clear Safari cache + uninstall the existing PWA if a prior Phase 12 test left one installed.
G. Confirm the GCP Web OAuth client has `https://mattmanNCF.github.io` in its Authorized JavaScript origins.

***

**Test 1 — MOB-AUTH-01 (incremental consent):**
  1. In desktop Settings -> Integrations -> Mobile Sync (Drive), click "Connect Google Drive".
  2. Browser opens with a Google consent prompt.
  3. **VERIFY**: The consent screen shows BOTH `calendar.events` AND `drive.appdata` scopes. A single consent, not a second browser prompt.
  4. Approve.
  5. Settings shows green health dot + "connected".
  - **PASS** if single prompt with both scopes. **FAIL** if two separate prompts or missing scope.

***

**Test 2 — MOB-PWA-01 (Pages install):**
  1. On iPhone, open Safari to https://mattmanNCF.github.io/notal-mobile/ .
  2. Page loads. Shows "Notal Capture" header.
  3. Safari address-bar Share button -> Add to Home Screen.
  4. PWA installs. Open from home screen.
  5. **VERIFY**: Opens in standalone mode (no browser chrome). Loads the capture textarea.
  - **PASS** if installed and standalone. **FAIL** if browser chrome appears or app fails to load.

***

**Test 3 — MOB-AUTH-01 + MOB-TRANS-01 + MOB-TRANS-02 + MOB-UX-01 (end-to-end capture):**
  1. Keep desktop Notal open (connected to Drive per Test 1).
  2. On the iPhone PWA (from Test 2), click "Connect Google".
  3. A Google consent prompt appears showing the `drive.appdata` scope.
  4. Approve. Badge turns to "connected".
  5. Type a note: "test-note-{timestamp}" (pick a unique string).
  6. Click Save. Observe badge progression: `local` -> `uploading` -> `on-drive`.
  7. On desktop, wait up to 60 seconds.
  8. **VERIFY**: The note appears in the desktop Notes tab within 60 seconds of the `on-drive` state on mobile.
  9. **VERIFY**: The desktop note's Submitted-At metadata or a source indicator shows it originated from mobile (source='mobile-drive' — surface via hover/DevTools if not shown in main UI).
  - **PASS** if note appears within 60s. **FAIL** if longer than 90s or note text is garbled/missing.

***

**Test 4 — MOB-TRANS-03 (Drive file deletion post-ingest):**
  1. After Test 3 succeeds, open https://drive.google.com from desktop browser.
  2. The file uploaded by Test 3 is NOT visible in regular My Drive (appDataFolder is hidden).
  3. Instead, in desktop Notal: open DevTools on the main window (View menu if exposed, or run `npm run dev` alternative session to query via IPC console). In Notal's main process log (file: `%APPDATA%/notal/logs/main.log` or similar), observe that the ingest loop deleted the file.
  4. Alternative automated check: run from desktop terminal:
     ```
     curl -H "Authorization: Bearer <access-token-from-oauth-playground-using-desktop-client>" "https://www.googleapis.com/drive/v3/files?spaces=appDataFolder&fields=files(id,name)"
     ```
     Expect: no file with the Test 3 note's `note-<timestamp>.json` name (it should have been deleted).
  - **PASS** if Drive listing does not contain the test note file. **FAIL** if the file persists.

***

**Test 5 — MOB-PWA-02 (capture-only UI):**
  1. Inspect the iPhone PWA.
  2. **VERIFY**: No wiki link, no search bar, no browse button, no tab navigation. Only a textarea, Save button, Connect/Disconnect button, and the local list of queued+sent notes.
  - **PASS** if UI is strictly capture + local list. **FAIL** if any browse/search/wiki element present.

***

**Test 6 — MOB-AUTH-02 (revoke surfacing):**
  1. On a desktop browser, visit https://myaccount.google.com/permissions .
  2. Find "Notal" (or the name Matt gave the GCP project).
  3. Click Remove Access / Revoke.
  4. Wait up to 2 minutes for the desktop Notal poll loop to attempt its next poll and fail with 401.
  5. Open desktop Settings -> Integrations -> Mobile Sync (Drive).
  6. **VERIFY**: Red message "Google has revoked access. Reconnect to resume mobile sync." is shown.
  - **PASS** if message appears. **FAIL** if state stays green or some other error surfaced.

***

**Test 7 — MOB-QUOTA-01 + MOB-UX-02:**
  7a. Reconnect Drive via Test 6's Reconnect button. Confirm green.
  7b. Induce MOB-UX-02 wake banner:
     1. Upload 3 test envelopes to appDataFolder via OAuth Playground using the Web client (same procedure as Plan 12-01 verify script).
     2. Quit Notal desktop (File -> Exit or tray -> Quit; NOT just close window — tray should also exit).
     3. Relaunch Notal.
     4. **VERIFY**: Green banner "Pulled 3 notes from mobile" appears briefly at the top of the window.
     - **PASS** if banner shows correct count. **FAIL** otherwise.
  7c. Induce MOB-QUOTA-01 warn:
     1. Upload a ~11MB file to appDataFolder (use Web client; a single big envelope is fine — ajv will reject it as too large and desktop will delete it, but the size check runs on file listing before deletion).
     2. Wait for next 30s UI refresh of DriveMobileSection.
     3. **VERIFY**: Amber "Warning: folder >10 MB" message appears under the quota gauge.
     - **PASS** if warning renders. **FAIL** otherwise.
  7d. Induce MOB-QUOTA-01 hard-stop (optional — skip if 10MB uploads are painful):
     1. Upload ~100+ MB of garbage to appDataFolder.
     2. **VERIFY**: Red "Folder >100 MB — ingestion halted." message appears. Desktop Notal ingest loop stops processing new files.
     - **PASS** if halt message shows and ingestion visibly stops. **FAIL** otherwise.
