**1. src/preload/index.d.ts — add `tags` to NoteRecord**

The existing NoteRecord interface (lines 1-9) is missing the `tags` field. The DB stores tags as JSON
text; the IPC handler already returns it. The renderer discards it because the type doesn't declare
the field. Add `tags: string[]` to the interface:

```typescript
interface NoteRecord {
  id: string
  rawText: string
  submittedAt: string
  aiState: 'pending' | 'complete' | 'failed'
  aiAnnotation: string | null
  organizedText: string | null
  aiInsights: string | null
  tags: string[]   // ← ADD THIS LINE
}
```

Note: this is declared `string[]` even though the DB returns a JSON string. The parse site is in
NotesTab (step 2 below) — NoteCard always receives a proper `string[]`.

**2. src/renderer/src/components/NotesTab.tsx — three sub-changes**

a) Extend the local NoteRecord interface to add `tags: string[]` (same addition as preload/index.d.ts).

b) Fix the `getAll()` init load (currently lines 20-29). Parse the JSON tags string into a proper
   `string[]` for every loaded note. The DB returns `tags` as a JSON string like `'["physics","math"]'`
   — NOT already an array. Replace the current `.then` handler with:

```typescript
window.api.notes.getAll()
  .then((loaded) => {
    setNotes(loaded.map(n => {
      let tags: string[] = []
      try { tags = JSON.parse((n as any).tags ?? '[]') } catch { /* leave empty */ }
      return { ...n, tags }
    }))
    setLoading(false)
  })
  .catch((err) => {
    console.error('Failed to load notes:', err)
    setLoading(false)
  })
```

c) Fix the `onAiUpdate` handler (currently line 34). The preload payload includes `tags` and `insights`
   but the current handler only destructures `{ noteId, aiState, aiAnnotation, organizedText }`.
   Note: the preload field is `insights` (not `aiInsights`) — confirmed in preload/index.d.ts line 26.
   Replace the current handler with:

```typescript
const unsub = window.api.onAiUpdate(({ noteId, aiState, aiAnnotation, organizedText, tags, insights }) => {
  setNotes((prev) =>
    prev.map((n) =>
      n.id === noteId
        ? {
            ...n,
            aiState: aiState as NoteRecord['aiState'],
            aiAnnotation,
            organizedText: organizedText ?? null,
            tags: tags ?? n.tags,
            aiInsights: insights ?? n.aiInsights,
          }
        : n
    )
  )
})
```

d) Add `tags: []` to the optimistic note in `handleSubmit` (the object currently missing the tags
   field, which would now cause a TypeScript error since NoteRecord requires it):

```typescript
const optimistic: NoteRecord = {
  id: `optimistic-${Date.now()}`,
  rawText,
  submittedAt: new Date().toISOString(),
  aiState: 'pending',
  aiAnnotation: null,
  organizedText: null,
  aiInsights: null,
  tags: [],   // ← ADD THIS
}
```
