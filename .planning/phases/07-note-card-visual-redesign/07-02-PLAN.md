**1. Add state and refs for expand behavior.**

After the existing state declarations (`menu`, `menuRef`) and the `cardRef` added in Task 1, add:
```typescript
const [expanded, setExpanded] = useState(false)
const [cardRect, setCardRect] = useState<DOMRect | null>(null)
const leaveTimer = useRef<ReturnType<typeof setTimeout> | null>(null)
```

**2. Replace the stub handleMouseEnter/handleMouseLeave with full implementations.**

Replace the stubs added in Task 1 with:
```typescript
const handleMouseEnter = useCallback(() => {
  if (leaveTimer.current) clearTimeout(leaveTimer.current)
  const rect = cardRef.current?.getBoundingClientRect()
  if (rect) setCardRect(rect)
  setExpanded(true)
}, [])

const handleMouseLeave = useCallback(() => {
  leaveTimer.current = setTimeout(() => setExpanded(false), 120)
}, [])
```

CRITICAL: `handleMouseEnter` MUST call `getBoundingClientRect()` fresh inside the callback —
not in a useEffect. The cursor is over the card at hover time, so the rect is correct at that
instant. Stale rects cause overlay offset after scrolling (Pitfall 5).

**3. Add cleanup useEffect for leaveTimer.**

Add this useEffect after the existing useEffects:
```typescript
useEffect(() => () => {
  if (leaveTimer.current) clearTimeout(leaveTimer.current)
}, [])
```

**4. Add the hover-expand portal overlay to the JSX.**

Add this block AFTER the existing context menu portal (the `{menu && createPortal(...)}` block).
The two portals are siblings — both render to `document.body`:

```tsx
{expanded && cardRect && createPortal(
  <div
    onMouseEnter={() => { if (leaveTimer.current) clearTimeout(leaveTimer.current) }}
    onMouseLeave={handleMouseLeave}
    style={{
      position: 'fixed',
      left: cardRect.left,
      top: cardRect.top,
      width: 300,
      maxHeight: 300,
      zIndex: 9998,
      overflowY: 'auto',
    }}
    className="rounded-sm bg-[#1f1f18] border border-white/10 shadow-2xl"
  >
    {/* User text — full, not truncated */}
    <div className="p-3 pb-2">
      <p className="text-sm text-gray-200 whitespace-pre-wrap leading-relaxed break-words">{note.rawText}</p>
    </div>

    {/* User/AI divider — only shown when AI content is present */}
    {(note.aiAnnotation || note.aiInsights) && (
      <div className="flex items-center gap-2 px-3 py-1">
        <div className="flex-1 border-t border-white/10" />
        <span className="text-[10px] text-gray-500">AI</span>
        <div className="flex-1 border-t border-white/10" />
      </div>
    )}

    {/* AI annotation */}
    {note.aiAnnotation && (
      <div className="px-3 pb-2">
        <p className="text-xs text-blue-400/70 leading-relaxed">{note.aiAnnotation}</p>
      </div>
    )}

    {/* AI insights — appended below annotation */}
    {note.aiInsights && (
      <div className="px-3 pb-3">
        <p className="text-xs text-gray-400 leading-relaxed">{note.aiInsights}</p>
      </div>
    )}
  </div>,
  document.body
)}
```

CRITICAL — overlay mouse handlers: The `onMouseEnter` on the overlay div MUST cancel `leaveTimer`.
Without this, moving the cursor from the compact card to the overlay triggers `handleMouseLeave`
on the card, starts the 120ms timer, and the overlay closes before the cursor reaches it. The
overlay's `onMouseEnter` cancels the timer so the overlay stays open. The overlay's `onMouseLeave`
restarts the timer so moving off the overlay closes it.

zIndex is 9998 — the context menu uses 9999 so context menus always appear above the expand overlay.

The overlay is positioned `fixed` at `cardRect.left / cardRect.top` — same top-left corner as the
compact card, so it appears to "lift" directly over the card rather than appearing offset. Combined
with the larger size (300px wide vs ~120–160px wide card), the card visually expands in place.
